# Roadmap inicial do Mini

## Passo 1 (feito)
- Estrutura unificada para Web + Flutter.
- Tela inicial unica com `mini.png` centralizado.

## Passo 2
- Definir contrato com backend (health, auth, perfil).
- Criar cliente API unico no Flutter e no Web.

## Passo 3
- Login infantil/ministerio com perfis (lider, professor, responsavel).
- Primeira area: cadastro de turmas.

## Passo 4
- Build e publicacao:
  - Web em `/mini`
  - Android (Play Store)
  - iOS (App Store)
